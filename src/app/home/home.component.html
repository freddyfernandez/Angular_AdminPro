Carrito de compras

Desarrollado en Visual Studio .Net, donde se implemento las funciones Action Result con sus respectivas sesiones,transacciones con SQlServer, Se crearon vistas parciales para la interaccion con los modulos del catálogo de producto hasta registrar el pedido

Funcionalidad

La estructura del diseño se realizo mediante HTML, CSS, JavaScript y jquery. La usabilidad es responsiva y adicionalmente tiene un diseño amigable para el reporte de ventas e inicio de sesion.

Panel de control

Para el acceso a los datos se realizo el consumo de servicios Api Rest y WCF al conectar con el controlador, aqui se creo el modulo de administración del sistema, en el cual podemos realizar el Mantenimiento y la validadcion de la base de datos Sql Server

Estructura MVC Strut2

Desarrollado en Eclipse Java, se implemento la solucion web con los frameworks Struts 2 y MyBatis en forma combinada.usando de la herramientas; Tomcat 9.0,IReport y se aplico la metodología MVC Struts 2. Gestionaremos la interacción con los datos mediante MYSQL,procedimientos almacenados,triggers. Los módulos creados son, la seccion por categorias de los productos, administración de clientes,trabajadores.socios y reporte de Pedidos.

El diseño incluyen Bootstrap, la gestion de la base de datos mediante sesiones por privilegios de administrador o socio, para actualizar nuestros registros, ademas el uso de Ajax para realizar busqueda por nombre o codigo y el manejo de archivos XML para interconectar las capas de las clases objetos,interfaces,action support y las vistas jsps.

Layouts

Desarrollado en Android Studio, el manejo de la interface mediante propiedades Layouts en los archivos XML,se hizo uso de clases Activity para el manejo de vistas y controladores, adicionalmente se añadió el material design entre iconos y colores en nuestra aplicación móvil. Se aplicó la metodología por capas. Realizaremos la ejecución con el JDK de JAVA. La estructura del diseño será manejada archivos XML mediante componentes Layout.

Navegacion

Los módulos creados son: Inicio de Sesión, Creación de Post, Creación Historia, y Edición de Perfil.manipulacion de imageens con la libreria picaso, y eventos para realizar notificaciones.

Almacenamiento y Servicios

Todos los datos son guardados en la nube en Firebase,el registro de de sesion mediante facebook y una seccion para geolocalizar la ubicacion de dispositivos moviles de google services.

Modelamiento DAO

Desarrollado en Java , el modelamiento de paquetes ordenados en vistas,servicios,clases, herencias, interfaces, factories y reutilizacion de librerias, para validar registros,configurar la fecha, consultar la base de datos.

Generacion de reportes basados transact sql,procedimientos almacenados,consultas anidadas, reporte en jasper report, uso de trigger para actualizar tablas de la base de datos.

Modelo MVC

Sistema de Ventas elaborado con Visual studio .Net, perzonalizable ya que esta programado bajo un paradigma de capas: datos,negocio y presentacion.

Estructura Cliente - Servidor

Desarrollado en Spring Tool Suite Java. Se aplico la metodología JPA (persistencia de datos). Almacen de datos alojado en hostinger con base de datos MYSQL, la web desplegada en heroku y el control de versiones con github. En la web servidor se desarrollo la estructura del servicio Api Rest. En la web cliente se implemento Los módulos de inicio de sesion para administradores o clientes, configuracion de cuenta o perfil, Gestion de Publicidad bancaria,transferencia entre cuentas.

Validacion de datos numerico,letras,longitud, datos vacios,etc desde el controlador spring, AJAX y javascript. Testeado en Junit y Rational Tester. Como resultado obtenemos las notificaciones en tiempo real de cada operacion realizada.
